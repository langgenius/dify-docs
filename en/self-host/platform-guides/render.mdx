Deploy Dify to [Render](https://render.com) using Infrastructure as Code with a `render.yaml` Blueprint. This guide walks you through deploying Dify's core services on Render's cloud platform.

## Prerequisites

- A [Render account](https://dashboard.render.com/register)
- A GitHub account (to fork the Dify repository)

## Architecture Overview

Dify on Render consists of the following services:

| Service | Type | Description |
|---------|------|-------------|
| `dify-api` | Web Service | Backend API server (Python/Flask) |
| `dify-worker` | Background Worker | Celery worker for async tasks |
| `dify-web` | Web Service | Frontend application (Next.js) |
| `dify-db` | PostgreSQL | Primary database |
| `dify-redis` | Redis | Cache and message broker |

> **Note:** The sandbox service and SSRF proxy are not included in this basic deployment. For production use with code execution features, additional configuration is required.

## Quick Deploy

### Option 1: One-Click Deploy

Click the button below to deploy Dify to Render:

[![Deploy to Render](https://render.com/images/deploy-to-render-button.svg)](https://render.com/deploy?repo=https://github.com/langgenius/dify)

### Option 2: Manual Blueprint Setup

1. Fork the [Dify repository](https://github.com/langgenius/dify) to your GitHub account

2. Create a `render.yaml` file in the root of your forked repository:

```yaml

    name: dify-api

        generateValue: true

          name: dify-web
          type: web
          property: host

        fromService:
          name: dify-api
          type: web
          property: host
      - key: APP_WEB_URL

          name: dify-web
          type: web
          property: host

        fromDatabase:
          name: dify-db
          property: user

        fromDatabase:
          name: dify-db
          property: password
      - key: DB_HOST
        fromDatabase:

          property: host

          type: redis

      - key: STORAGE_TYPE
        value: s3
      - key: VECTOR_STORE
        value: pgvector
      - key: PGVECTOR_HOST
        fromDatabase:
          name: dify-db
          property: host
      - key: PGVECTOR_PORT
        fromDatabase:

      mountPath: /app/api/storage
      sizeGB: 10

  # Dify Web Frontend
  - type: web
    name: dify-web
    runtime: image
    image:
      url: docker.io/langgenius/dify-web:latest
    plan: starter
    envVars:
      - key: CONSOLE_API_URL

    maxmemoryPolicy: allkeys-lru

    plan: starter
    postgresMajorVersion: 15
    ipAllowList: []
```

3. Go to the [Render Dashboard](https://dashboard.render.com)

4. Click **New** → **Blueprint**

5. Connect your GitHub account and select your forked Dify repository

6. Render will detect the `render.yaml` and create all services automatically

## Post-Deployment Setup

### 1. Enable pgvector Extension

After deployment, connect to your PostgreSQL database and enable the pgvector extension:

```sql
CREATE EXTENSION IF NOT EXISTS vector;
```

You can do this via the Render Dashboard:
1. Go to your `dify-db` database
2. Click **Shell** tab
3. Run the SQL command above

### 2. Access Dify

Once all services are running:

1. Find your `dify-web` service URL in the Render Dashboard
2. Navigate to `https://your-dify-web-url.onrender.com/install`
3. Set up your admin account using the `INIT_PASSWORD` from your environment variables

### 3. Configure Storage (Recommended)

For production deployments, configure S3-compatible storage:

1. Go to your `dify-api` service in the Render Dashboard
2. Add the following environment variables:

```
S3_ENDPOINT=your-s3-endpoint
S3_BUCKET_NAME=your-bucket-name
S3_ACCESS_KEY=your-access-key
S3_SECRET_KEY=your-secret-key
S3_REGION=your-region
```

## Environment Variables

Key environment variables you may want to customize:

| Variable | Description | Default |
|----------|-------------|---------|
| `SECRET_KEY` | Application secret key | Auto-generated |
| `INIT_PASSWORD` | Initial admin password | Auto-generated |
| `LOG_LEVEL` | Logging level | `INFO` |
| `VECTOR_STORE` | Vector database type | `pgvector` |
| `STORAGE_TYPE` | File storage type | `local` |

For a complete list of environment variables, see the [Environment Variables documentation](/en/self-host/configuration/environments).

## Scaling

### Horizontal Scaling

To handle more traffic:

1. Go to your service in the Render Dashboard
2. Click **Settings** → **Instance Type**
3. Upgrade to a higher plan or increase instance count

### Database Scaling

For larger datasets:

1. Go to your `dify-db` database
2. Click **Settings** → **Plan**
3. Upgrade to a higher tier

## Troubleshooting

### Services Not Starting

1. Check the **Logs** tab for each service in the Render Dashboard
2. Verify all environment variables are correctly set
3. Ensure database connections are established

### Database Connection Issues

1. Verify the database is in a `Running` state
2. Check that the pgvector extension is enabled
3. Confirm IP allow list settings if using private networking

### Storage Issues

If file uploads fail:
1. Verify disk is properly mounted
2. Check storage permissions
3. Consider switching to S3 storage for production

## Upgrading Dify

To upgrade to a new version:

1. Update the image tags in your `render.yaml`:

```yaml
image:
  url: docker.io/langgenius/dify-api:1.13.0  # Specify version
```

2. Commit and push the changes
3. Render will automatically redeploy with the new version

Alternatively, trigger a manual deploy from the Render Dashboard.

## Limitations

- **Sandbox Service**: Code execution sandbox is not included in this basic setup
- **SSRF Proxy**: Not included; external API calls may have different security characteristics
- **Plugin Daemon**: Requires additional configuration for plugin support

For a full-featured deployment with all services, consider using Docker Compose on a dedicated server or Kubernetes.
