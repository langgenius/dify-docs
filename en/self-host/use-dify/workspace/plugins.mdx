Integrations connect Dify to the outside world. They supply the model providers that power your apps, the tools your agents and workflows call, the data sources behind your knowledge bases, and the services that trigger or extend what you build.

Install an integration once, and every app in your workspace can use it. Browse and manage them all from **Integrations**.

## Integration Types

- [**Model Provider**](/en/self-host/use-dify/workspace/model-providers): The models that power your apps, from LLMs to embedding and rerank models.

- [**Tool**](/en/self-host/use-dify/workspace/tools): Capabilities your agents and workflows can call, including built-in tool plugins, MCP servers, published workflows, and OpenAPI services.

- [**Data Source**](/en/self-host/use-dify/knowledge/knowledge-pipeline/authorize-data-source): External content for knowledge bases built with the Knowledge Pipeline, such as Notion pages or websites.

- [**Trigger**](/en/self-host/use-dify/nodes/trigger/plugin-trigger): Start a workflow automatically when an external event fires.

- **Agent Strategy**: The reasoning strategies an [Agent node](/en/self-host/use-dify/nodes/agent) uses to plan and call tools.

- [**Custom Endpoint**](/en/self-host/use-dify/workspace/api-extension/api-extension): Connect your own API to add content moderation or external data tools to your apps.

- **Extension**: Integrate external services into your apps using HTTP webhooks.

## Install Integrations

Add integrations from three sources:

- [**Marketplace**](https://marketplace.dify.ai/): Official and partner integrations, tested and maintained.
- **GitHub**: Install from any public repository by URL and version.
- **Local upload**: Custom `.zip` packages for private or internal integrations.

<Tip>
  On Dify Enterprise, admins can restrict installation across the deployment, limiting it to official integrations, official plus verified partners, or marketplace installs only. [Contact sales](https://udify.app/chat/QuwcpW1oBNcfeL55) to learn more.
</Tip>

## Permissions and Updates

The Workspace Owner and Admins set two workspace-wide permissions for integrations, each to **Everyone**, **Admins**, or **No one**:

- **Who can install and manage integrations**: Installing, upgrading, and removing. Defaults to **Everyone**.
- **Who can debug integrations**: Access to the debugging key. Defaults to **No one**.

You can also configure integrations to update automatically. For each category, such as tools or model providers, choose an update strategy (off, patch versions only, or always the latest) and whether it applies to every integration or a chosen subset.

After installation, most integrations need configuration such as API keys or endpoints, which apply across the workspace.
