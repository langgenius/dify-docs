title: Completion App API
sidebarTitle: Overview
description: API for Text Generator apps, covering completion messages, file uploads, voice, and feedback

Text Generator apps run in `completion` mode: each call submits its own input values and returns one response, with no conversation state carried between calls. This fits one-shot tasks like translation, article writing, and summarization, where every request stands on its own.

## Generate Text

[Send Completion Message](/en/api-reference/completion-messages/send-completion-message) fills the app's prompt template with the values in `inputs` and generates a response. Set `response_mode` to `streaming` for a live feed of `message` events ending in `message_end`, plus `tts_message` events when auto-play is enabled, or `blocking` to wait for the complete result in one response.

[Stop Completion Message Generation](/en/api-reference/completion-messages/stop-completion-message-generation) cancels a run using its `task_id`, but only works in streaming mode; a blocking call has already finished by the time it returns.

## Handle Files

[Upload File](/en/api-reference/files/upload-file) uploads an image, document, audio, or video file for [Send Completion Message](/en/api-reference/completion-messages/send-completion-message) to reference, scoped to the uploading end user. [Download File](/en/api-reference/files/download-file) previews or downloads a file uploaded that way, as long as it belongs to a message in your app.

When a response returns an end-user ID instead of full details, such as `created_by` in the Upload File response, [Get End User Info](/en/api-reference/end-users/get-end-user-info) resolves it.

## Transcribe and Synthesize Speech

[Convert Audio to Text](/en/api-reference/audio/convert-audio-to-text) transcribes an uploaded audio file (MP3, M4A, WAV, AMR, or MPGA, up to 30 MB) so end users can speak their input instead of typing it. [Convert Text to Audio](/en/api-reference/audio/convert-text-to-audio) synthesizes the generated response back into speech.

## Retrieve App Info and Settings

[Get App Info](/en/api-reference/applications/get-app-info) returns the app's name, description, tags, and mode. [Get App Parameters](/en/api-reference/applications/get-app-parameters) returns its input form configuration: feature switches, input parameter names, types, and defaults, useful for building a client UI.

[Get App Meta](/en/api-reference/applications/get-app-meta) returns tool icons and other configuration metadata. [Get App WebApp Settings](/en/api-reference/applications/get-app-webapp-settings) returns the WebApp's site configuration, theme, and customization options.

## Collect Feedback

End users rate a response with [Submit Message Feedback](/en/api-reference/feedback/submit-message-feedback) as `like` or `dislike`, with an optional text comment; pass `null` for `rating` to revoke a previous rating. [List App Feedbacks](/en/api-reference/feedback/list-app-feedbacks) returns every feedback submitted across the app, from both end users and admins.
