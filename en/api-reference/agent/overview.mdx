New Agent apps run in `agent` mode, following the same chat message flow as other chat-style apps: send a message, stream the reply, and manage conversations through shared endpoints. The model reasons and calls tools autonomously, but the stream exposes none of that—only plain `message` events.

<Note>
The New Agent app type is in beta. File inputs on [Send Chat Message](/en/api-reference/chat-messages/send-chat-message) are not yet supported for New Agent apps.
</Note>

## Send Messages and Stream Replies

[Send Chat Message](/en/api-reference/chat-messages/send-chat-message) sends a query to your New Agent app. New Agent apps support `streaming` only; `blocking` returns a 400 `bad_request` error. The stream's `message_end` event carries `usage` in its `metadata`—plus `annotation_reply` when the query matches an annotation—but never `retriever_resources`.

To interrupt a streaming reply before it finishes, call [Stop Chat Message Generation](/en/api-reference/chat-messages/stop-chat-message-generation).

## Manage Conversations

[List Conversations](/en/api-reference/conversations/list-conversations) returns the current user's conversations, ordered by most recent activity. [List Conversation Messages](/en/api-reference/conversations/list-conversation-messages) returns one conversation's message history for a scrolling chat UI; since the model's reasoning isn't exposed, each message's `agent_thoughts` array is always empty.

## Upload Files

[Upload File](/en/api-reference/files/upload-file) uploads an image, document, audio, or video file, scoped to the uploading end user. New Agent apps don't support this yet: file references passed to [Send Chat Message](/en/api-reference/chat-messages/send-chat-message) are accepted but ignored.

## Retrieve App Info and Configuration

[Get App Info](/en/api-reference/applications/get-app-info) returns the app's name, description, tags, and mode. [Get App Parameters](/en/api-reference/applications/get-app-parameters) returns its input form configuration: feature switches, input parameter names, types, and defaults, useful for building a client UI.

[Get App Meta](/en/api-reference/applications/get-app-meta) returns tool icons and other configuration metadata.
