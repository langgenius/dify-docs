New Agent apps run in `agent` mode, following the same chat message flow as other chat-style apps: send a message, stream the reply, and manage conversations through shared endpoints.

The model reasons and calls tools autonomously, but the stream exposes none of that—only plain `message` events. The `message_end` event's `metadata` carries `usage` (plus `annotation_reply` when the query matches an annotation) but never `retriever_resources`.

<Note>
The New Agent app type is in beta.
</Note>

## Send Messages and Stream Replies

- **[Send Chat Message](/en/api-reference/chat-messages/send-chat-message)**: send a query to your New Agent app. Streaming mode only; blocking mode returns a 400 `bad_request` error.
- **[Stop Chat Message Generation](/en/api-reference/chat-messages/stop-chat-message-generation)**: interrupt a streaming reply before it finishes.

## Manage Conversations

- **[List Conversations](/en/api-reference/conversations/list-conversations)**: the current user's conversations, ordered by most recent activity.
- **[List Conversation Messages](/en/api-reference/conversations/list-conversation-messages)**: one conversation's message history, for a scrolling chat UI. The model's reasoning isn't exposed here either: each message's `agent_thoughts` array is always empty.

## Upload Files

- **[Upload File](/en/api-reference/files/upload-file)**: upload an image, document, audio, or video file, scoped to the uploading end user. New Agent apps don't support file inputs yet: file references passed to [Send Chat Message](/en/api-reference/chat-messages/send-chat-message) are accepted but ignored.

## Retrieve App Info and Settings

- **[Get App Info](/en/api-reference/applications/get-app-info)**: the app's name, description, tags, and mode.
- **[Get App Parameters](/en/api-reference/applications/get-app-parameters)**: the input form configuration (feature switches, parameter names, types, and defaults), useful for building a client UI.
- **[Get App Meta](/en/api-reference/applications/get-app-meta)**: tool icons and other configuration metadata.
