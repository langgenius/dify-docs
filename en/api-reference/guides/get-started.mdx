Every app you publish in Dify doubles as a REST API, and so do your knowledge bases. You call the same app your end users interact with, from your own backend, with an API key.

## Get an API Key

For an app, open it and click **API Access** in the left sidebar, then create an API key. The key is scoped to that one app.

For knowledge bases, open the **Knowledge** page and click **Service API** in the top-right corner. A knowledge API key is broader: it can reach every knowledge base visible to the account that created it, so treat it with extra care — see the [Knowledge API](/en/api-reference/guides/knowledge) guide.

<Warning>
Call the API from your backend only. A key embedded in frontend code or a client app can be extracted and abused.
</Warning>

## Make Your First Call

Every request carries the key as a Bearer token. The base URL is `https://api.dify.ai/v1`; for self-hosted deployments, use your own instance's API base URL. Try [Get App Info](/en/api-reference/applications/get-app-info) — it works for every app type:

```bash
curl https://api.dify.ai/v1/info \
  -H "Authorization: Bearer $DIFY_API_KEY"
```

A response like this confirms the key works and shows which app it belongs to:

```json
{
  "name": "My Chat App",
  "description": "A helpful assistant",
  "mode": "chat",
  "tags": ["assistant"]
}
```

## Find Your App Type's Endpoints

What you can call next depends on the app type behind your key. Each app type has a page walking its full API surface: [Chatflow](/en/api-reference/guides/chatflow), [Workflow](/en/api-reference/guides/workflow), [New Agent](/en/api-reference/guides/agent), [Knowledge](/en/api-reference/guides/knowledge), [Chatbot and Agent](/en/api-reference/guides/chat), and [Text Generator](/en/api-reference/guides/completion).

Every endpoint page in the reference lists which app types it serves, so you can also start from any endpoint and check whether it applies to you.
