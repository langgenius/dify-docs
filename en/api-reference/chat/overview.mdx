Chatbot apps (`chat` mode) and Agent apps (`agent-chat` mode) share this API surface. The same endpoints send messages, manage conversations, handle files, and retrieve app settings; only the streamed reply differs between the two.

## Send Messages and Stream Replies

[Send Chat Message](/en/api-reference/chat-messages/send-chat-message) sends a query to your Chatbot or Agent app. Chatbot apps stream the reply as `message` events; Agent apps stream `agent_thought` and `agent_message` events instead. `blocking` mode isn't available for Agent apps, only `streaming`.

To interrupt a streaming reply before it finishes, call [Stop Chat Message Generation](/en/api-reference/chat-messages/stop-chat-message-generation). After a reply completes, [Get Next Suggested Questions](/en/api-reference/chat-messages/get-next-suggested-questions) proposes follow-up questions based on the conversation so far.

## Manage Conversations

[List Conversations](/en/api-reference/conversations/list-conversations) returns the current user's conversations, ordered by most recent activity. [List Conversation Messages](/en/api-reference/conversations/list-conversation-messages) returns one conversation's message history for a scrolling chat UI. [Rename Conversation](/en/api-reference/conversations/rename-conversation) sets or auto-generates a display name, and [Delete Conversation](/en/api-reference/conversations/delete-conversation) removes a conversation.

Conversations created through the API stay isolated from conversations started in the app's WebApp.

Conversation variables persist values across turns in a session. [List Conversation Variables](/en/api-reference/conversations/list-conversation-variables) reads their current values, and [Update Conversation Variable](/en/api-reference/conversations/update-conversation-variable) changes one directly through the API.

## Handle Files

[Upload File](/en/api-reference/files/upload-file) uploads an image, document, audio, or video file for [Send Chat Message](/en/api-reference/chat-messages/send-chat-message) to reference, scoped to the uploading end user. [Download File](/en/api-reference/files/download-file) previews or downloads a file uploaded that way, as long as it belongs to a message in your app.

When a response returns an end-user ID instead of full details, such as `created_by` in the Upload File response, [Get End User Info](/en/api-reference/end-users/get-end-user-info) resolves it.

## Transcribe and Synthesize Speech

[Convert Audio to Text](/en/api-reference/audio/convert-audio-to-text) transcribes an uploaded audio file (MP3, M4A, WAV, AMR, or MPGA, up to 30 MB) so end users can speak their input instead of typing it. [Convert Text to Audio](/en/api-reference/audio/convert-text-to-audio) synthesizes a reply back into speech.

## Retrieve App Info and Settings

[Get App Info](/en/api-reference/applications/get-app-info) returns the app's name, description, tags, and mode. [Get App Parameters](/en/api-reference/applications/get-app-parameters) returns its input form configuration: feature switches, input parameter names, types, and defaults, useful for building a client UI.

[Get App Meta](/en/api-reference/applications/get-app-meta) returns tool icons and other configuration metadata. [Get App WebApp Settings](/en/api-reference/applications/get-app-webapp-settings) returns the WebApp's site configuration, theme, and customization options.

## Collect Feedback

End users rate a reply with [Submit Message Feedback](/en/api-reference/feedback/submit-message-feedback) as `like` or `dislike`, with an optional text comment; pass `null` for `rating` to revoke a previous rating. [List App Feedbacks](/en/api-reference/feedback/list-app-feedbacks) returns every feedback submitted across the app, from both end users and admins.

## Manage Annotations

Annotations pair a question with a fixed answer that the app returns directly instead of generating a new response. Manage them with [Create Annotation](/en/api-reference/annotations/create-annotation), [List Annotations](/en/api-reference/annotations/list-annotations), [Update Annotation](/en/api-reference/annotations/update-annotation), and [Delete Annotation](/en/api-reference/annotations/delete-annotation).

[Configure Annotation Reply](/en/api-reference/annotations/configure-annotation-reply) turns annotation matching on or off. It runs asynchronously, so poll [Get Annotation Reply Job Status](/en/api-reference/annotations/get-annotation-reply-job-status) with the returned job ID to confirm it finished.
