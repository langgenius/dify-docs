A resource access rule grants a permission set (a named bundle of resource permissions such as *Can edit* or *Can preview*) to specific roles or members. You manage these rules in **Settings** > **Resource Access**.

Resource access rules set who can access each app and knowledge base, and what they can do with it. To control who can create apps and knowledge bases, go to **Settings** > [**Roles & Permissions**](/en/use-dify/workspace/roles-and-permissions).

## How Resource Access Rules Apply

The rules you set in **Settings** > **Resource Access** are workspace defaults. Every new app and knowledge base inherits these defaults and stays in sync as you change them.

When an app or knowledge base needs access that differs from the defaults, you can override the rules in its own access settings, changing which roles or members each permission set applies to. From then on, that resource stops syncing with the workspace defaults.

When several rules grant a role or member access to the same resource, their permissions *combine*. For example, a member who gets *Can preview* from one rule and *Can edit* from another can do anything either set allows.

## Permission Set Types

Permission sets come in two types, system and custom. Apps and knowledge bases each have their own sets.

- **System permission sets**

    Built-in sets like *Full access*, *Can edit*, and *Can preview*. They can't be edited or deleted, but you can duplicate one and edit the copy, or change which roles or members they apply to.

- **Custom permission sets**

    Additional sets that fit your needs. A new workspace starts with any custom sets your system administrator has predefined in the admin console.

    Deleting a custom permission set detaches it from every role, member, and resource it was bound to.

## Manage Resource Access Rules

To create a new access rule for apps or knowledge bases, click **New permission set**, then name it and choose the permissions it includes.

To start from an existing set, click its **...** > **Duplicate** and edit the copy.

To grant a set to people, select **+ Add** on it and pick from the **Roles** or **Members** tabs:

- **Binding a role** grants the set to everyone who holds that role.

- **Binding a member** grants the set to that one person regardless of their roles.
