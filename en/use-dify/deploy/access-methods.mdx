After deploying an app, you can choose how users reach it. There are three access methods: the WebApp, the Dify CLI, and API tokens.

- **WebApp and Dify CLI**: you control who can use the app by choosing who is allowed in.
- **API tokens**: credentials for programmatic calls. Access depends on who holds the token, not on who you allow.

Each method is off by default. Turn one on for the deployment, then configure its settings per environment.

## WebApp

The WebApp is a hosted page where end users can use your deployed app. Turn it on, and each environment gets its own shareable URL.

Choose who can open the WebApp in each environment:

- **Anyone with the link**: anyone can use it without signing into Dify.
- **All members within the platform**: everyone in the platform.
- **Specific members within the platform**: only the roles or members you select in the platform.

By default, all platform members have access.

## Dify CLI

The Dify CLI lets a person or an agent call your deployed app from the command line.

The Dify CLI uses the same per-environment access permissions as the WebApp.

## API Tokens

API tokens let a program or service call your deployed app over HTTP. Each token is scoped to one environment.

Turn on **API Tokens** for the deployment, then generate a token for an environment. Copy it when it appears, since it is shown only once. 

You can revoke a token at any time, which stops it from working immediately.
