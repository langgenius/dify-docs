To deploy an app, you create a deployment once, then manage everything from there: building releases, deploying them to environments, and updating or rolling back what each environment runs.

<Info>

- To understand the underlying concepts like deployments and releases, see [Overview](/en/use-dify/deploy/overview).
- To control who can reach your deployed app and how, see [Configure Access Methods](/en/use-dify/deploy/access-methods).

</Info>

## Create a Deployment

In **Deployments**, click **New** and follow the guided flow to select a published Workflow app and build a release from it. You can then deploy the release to an environment right away, or skip this step and deploy later.

Publish the app in **Studio** before creating the deployment, since releases are built from its latest published version.

## Deploy a New Release

Each time you publish a new version of the app in **Studio**, ship the change by creating a new release and deploying it:

1. In a deployment, go to **Releases** and click **Create Release**.

2. Fill in the release details, then click **Create**.

3. Open the created release's action menu (`...`), then deploy it to an available environment.

You can also export a release as a DSL file for backup or reuse elsewhere.

## Deploy to More Environments

A release can run in several environments at the same time, for example the same release serving both `staging` and `production`.

To deploy a release to another environment, use either method: 

- In **Releases**, open the release's action menu (`...`) and deploy it to a new environment.
- In **Instances**, click **Deploy to New Environment**, then select the release and a target environment.

## Update or Roll Back an Instance

Each instance runs one active release at a time. To update it with a newer release or roll it back to an earlier release, use either method:

- In **Instances**, open the instance's action menu (`...`) and click **Deploy Another Release**.
- In **Releases**, open a release's action menu and deploy it to the environment running that instance.

<Warning>
If a deployment fails, the instance does not automatically revert to its previous release. To roll back, manually deploy a previous release.
</Warning>

To stop serving your app in an environment, choose **Undeploy** from the instance's action menu in **Instances**. End-user access stops immediately, and you can redeploy a release later.
