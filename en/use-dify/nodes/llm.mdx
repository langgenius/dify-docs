## Introduction

The LLM node calls a large language model (LLM) to generate responses based on your instructions and inputs from upstream nodes.

You can also **inject context** to build RAG applications, enable **conversation memory** (in Chatflows), force **structured JSON outputs** for reliable data extraction, let the LLM **call external tools** to complete complex tasks, etc.

## Choose the Right Model

Choose a model that best fits your task from configured model providers. The tags displayed on the model card can help you decide.

![Displayed Model Tags]()

|Tag|Description|
|:-|:-|
|`8K` / `32k` / `128k`, etc<br></br><br></br>(Context window)| Indicates how much text the model can process at once, including your prompt, injected context, conversation history, and the model's output.|
|`Chat` / `Completion`<br></br><br></br>(Model type)| <ul><li>`Chat`: Best for handling conversations and following complex instructions. |`Vision` / `Audio`/ `Video` / `Document`<br></br><br></br>(Multimodal capabilities)| Indicates which input types the model can understand directly. `Vision` indicates image input.|
|`Tool Call`| Indicates whether the model can natively call configured tools to complete complex tasks.<Info>Models without native support can still call tools through prompt engineering, though results may be less reliable.</Info>|

After selecting the model, you can adjust model parameters to control how it generates responses. Available parameters and presets vary by model.

## Add Context

Use `Context` to provide the model with additional information for more accurate and relevant responses, reducing hallucination.

The most common use case is passing the retrieval results from an upstream Knowledge Retrieval node for Retrieval-Augmented Generation (RAG).

To successfully inject context:

1. In the `Context` field, select the variable you want to use as context.

2. In your prompt, reference the `Context` variable.

<Tip>
    - When using context variables from knowledge retrieval, Dify automatically tracks citations so users see information sources.
    
    - Learn more about [using knowledge retrieval results as LLM context](/en/use-dify/nodes/knowledge-retrieval#use-with-llm-nodes).
</Tip>

## Enable Conversation Memory (Chatflows Only)

Toggle on `Memory` to include recent chat history in prompts, so the model can answer follow-up questions more coherently and with better context within a conversation.

Adjust **Window Size** to control how many recent exchanges to retain. For example, `5` keeps the last 5 user-message and LLM-response pairs.

<Info>
    Once memory is enabled, a **User** prompt is automatically added to pass the current user query and any uploaded files.
</Info>

<Note>
    Memory is node-specific and doesn't persist between different conversations.
</Note>

## Use External Tools

Add tools to enable the model to interact with external services to complete tasks that require real-time data or actions beyond text generation, such as searching the web, querying databases, or calling APIs.

Adjust **Max Iterations** to limit how many times the model can call tools in a single response. This helps control latency and cost while preventing excessively long tool loops.

**Native vs. Prompt-based Tool Calling**

- Models **with native support** (indicated by a `Tool Call` tag) automatically decide when to use a tool, format the request correctly, and interpret the results.

- Models **without native support** can still use tools via prompt engineering. When you add a tool to such models, you'll be prompted to apply a ReAct prompt, which guides the model to use tools step by step.

## Analyze Image Inputs

Toggle on **Vision** to let the model interpret and analyze images directly. Only available for models with a `Vision` tag.

Select an image variable and adjust **Resolution** to control the detail level at which the model processes the image.

- **High**: Better accuracy for complex images but uses more tokens

- **Low**: Faster processing with fewer tokens for simple images

## Force Structured Output

Force the model to generate responses in a specific JSON object format, making the output easier for downstream processing.

1. Toggle on **Structured** next to **Output Variables**. The `structured_output` variable will appear at the end of the output variable list.

2. Click **Configure** to define the output schema using one of the four methods.

    - **Visual Editor**: Define simple structures with a no-code interface. The corresponding JSON schema is generated automatically.

    - **JSON Schema**: Directly write schemas for complex structures with nested objects, arrays, or validation rules.

    - **AI Generation**: Describe needs in natural language and let AI generate the schema.

    - **JSON Import**: Paste an existing JSON object to automatically generate the corresponding schema.

<Tip>
    Mark essential fields as **Required**. The node will error out if any required field is missing from the LLM's output.
</Tip>

<Note>
    For models without native JSON support, Dify includes the schema in the prompt, but strict adherence is not guaranteed.
</Note>

## Handle Errors

Configure **automatic retries for temporary issues** (like network glitches) and a fallback **error handling strategy** to keep the workflow running **if errors persist**.
