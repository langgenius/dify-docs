## 公開の仕組み

アプリを公開すると、Dify はすべてのチャネルにわたって最新の設定を有効化します。舞台裏で起こることは以下の通りです：

- **Web アプリ** は新機能と応答で即座に更新
- **API エンドポイント** は最新のモデルとワークフロー設定を提供
- **埋め込みウィジェット** は更新されたスタイルと動作を反映
- **MCP サーバー** は現在のアプリ機能へのアクセスを提供

<Warning>
公開すると、ライブアプリが現在の設定に置き換わります。ユーザーは次のインタラクションで即座に変更を確認できます。
</Warning>

## 公開のベストプラクティス

アプリを共有する前に、これらの設定を構成済みであることを確認してください：

- **アプリの説明** - ユーザーがアプリの機能を理解するのに役立つ
- **アイコンとブランディング** - アプリを認識しやすく専門的にする
- **アクセス制御** - アプリが公開であるべきか認証が必要かを決定
- **レート制限** - アプリの過度な使用から保護（API アクセスでは特に重要）

<Check>
すべての公開方法は同じアプリ設定を使用します。一度設定すれば、どこでも公開できます。
</Check>
