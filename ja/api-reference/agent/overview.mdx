New Agent アプリは `agent` モードで動作します。他のチャット形式アプリと同じく、メッセージを送信し、回答をストリーミングで受け取ります。会話も共通のエンドポイントで管理します。モデルは自律的に推論し、ツールを使用します。ただし、その過程は API 上に公開されません。ストリームに含まれるのは、通常の `message` イベントのみです。

<Note>
New Agent アプリタイプは現在ベータ版です。[チャットメッセージを送信](/ja/api-reference/chat-messages/send-chat-message) では、New Agent アプリへのファイル入力は未対応です。
</Note>

## メッセージの送信とストリーミング返信

[チャットメッセージを送信](/ja/api-reference/chat-messages/send-chat-message) は、New Agent アプリにリクエストを送信します。New Agent アプリは `streaming` のみに対応しています。`blocking` を指定すると 400 `bad_request` が返されます。ストリームの `message_end` イベントは `metadata` に `usage` を含みます。アノテーションが一致した場合は `annotation_reply` も含まれます。ただし `retriever_resources` は含まれません。

返信が完了する前に中断するには、[生成を停止](/ja/api-reference/chat-messages/stop-chat-message-generation) を呼び出してください。

## 会話の管理

[会話一覧を取得](/ja/api-reference/conversations/list-conversations) は、最終アクティビティの新しい順に現在のユーザーの会話一覧を返します。[会話履歴メッセージ一覧を取得](/ja/api-reference/conversations/list-conversation-messages) は、1 つの会話のメッセージ履歴をスクロール読み込み形式で返します。モデルの推論は公開されないため、各メッセージの `agent_thoughts` 配列は常に空になります。

## ファイルのアップロード

[ファイルをアップロード](/ja/api-reference/files/upload-file) は、画像、ドキュメント、音声、動画ファイルをアップロードします。アップロードしたエンドユーザー自身のみが利用できます。New Agent アプリはファイル入力に未対応です。[チャットメッセージを送信](/ja/api-reference/chat-messages/send-chat-message) にファイル参照を渡しても、受け付けられるだけで使用されません。

## アプリ情報と設定

[アプリケーションの基本情報を取得](/ja/api-reference/applications/get-app-info) は、アプリの名前、説明、タグ、モードを返します。[アプリケーションのパラメータ情報を取得](/ja/api-reference/applications/get-app-parameters) は、機能スイッチや入力パラメータの名前・型・デフォルト値といった入力フォームの設定を返します。クライアント UI の構築に役立ちます。

[アプリケーションのメタ情報を取得](/ja/api-reference/applications/get-app-meta) は、ツールアイコンなどの設定メタデータを返します。
