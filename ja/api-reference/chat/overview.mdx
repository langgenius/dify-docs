チャットボットアプリ（`chat` モード）と Agent アプリ（`agent-chat` モード）は、この API を共有します。同じエンドポイントでメッセージの送信、会話の管理、ファイルの操作、アプリ設定の取得を行います。両者で異なるのは、返信のストリーミングイベントのみです。

## メッセージの送信とストリーミング返信

[チャットメッセージを送信](/ja/api-reference/chat-messages/send-chat-message) は、チャットボットまたは Agent アプリにリクエストを送信します。チャットボットアプリは回答を `message` イベントとしてストリーミングします。Agent アプリでは、代わりに `agent_thought` と `agent_message` イベントが返されます。Agent アプリは `blocking` モードに対応しておらず、`streaming` のみ使用できます。

返信が完了する前に中断するには、[生成を停止](/ja/api-reference/chat-messages/stop-chat-message-generation) を呼び出してください。返信の完了後は、[次の推奨質問を取得](/ja/api-reference/chat-messages/get-next-suggested-questions) がそれまでの会話内容から次の質問を提案します。

## 会話の管理

[会話一覧を取得](/ja/api-reference/conversations/list-conversations) は、最終アクティビティの新しい順に現在のユーザーの会話一覧を返します。[会話履歴メッセージ一覧を取得](/ja/api-reference/conversations/list-conversation-messages) は、1 つの会話のメッセージ履歴をスクロール読み込み形式で返します。[会話の名前を変更](/ja/api-reference/conversations/rename-conversation) は表示名を設定または自動生成します。[会話を削除](/ja/api-reference/conversations/delete-conversation) は会話を削除します。

API 経由で作成した会話は、アプリの WebApp で開始した会話とは分離されます。

会話変数はセッション内の複数ターンにわたって値を保持します。[会話変数の取得](/ja/api-reference/conversations/list-conversation-variables) で現在の値を読み取り、[会話変数を更新](/ja/api-reference/conversations/update-conversation-variable) で API から直接値を変更できます。

## ファイルの操作

[ファイルをアップロード](/ja/api-reference/files/upload-file) は、画像、ドキュメント、音声、動画ファイルをアップロードし、[チャットメッセージを送信](/ja/api-reference/chat-messages/send-chat-message) から参照できるようにします。アップロードしたエンドユーザー自身のみが利用できます。[ファイルをダウンロード](/ja/api-reference/files/download-file) は、そのファイルをプレビューまたはダウンロードします。対象はアプリ内のメッセージに紐づくファイルに限られます。

アップロードファイルのレスポンスの `created_by` のように、レスポンスにはエンドユーザー ID のみが含まれることがあります。その場合は、[エンドユーザー取得](/ja/api-reference/end-users/get-end-user-info) で詳細情報を取得できます。

## 音声とテキストの相互変換

[音声をテキストに変換](/ja/api-reference/audio/convert-audio-to-text) は、アップロードされた音声ファイル（MP3、M4A、WAV、AMR、MPGA のいずれか、最大 30 MB）をテキストに変換します。エンドユーザーは入力をテキストではなく音声で行えます。[テキストを音声に変換](/ja/api-reference/audio/convert-text-to-audio) は、返信を音声として合成します。

## アプリ情報と設定の取得

[アプリケーションの基本情報を取得](/ja/api-reference/applications/get-app-info) は、アプリの名前、説明、タグ、モードを返します。[アプリケーションのパラメータ情報を取得](/ja/api-reference/applications/get-app-parameters) は、機能スイッチや入力パラメータの名前・型・デフォルト値といった入力フォームの設定を返します。クライアント UI の構築に役立ちます。

[アプリケーションのメタ情報を取得](/ja/api-reference/applications/get-app-meta) は、ツールアイコンなどの設定メタデータを返します。[アプリの WebApp 設定を取得](/ja/api-reference/applications/get-app-webapp-settings) は、WebApp のサイト設定、テーマ、カスタマイズ項目を返します。

## フィードバックの収集

エンドユーザーは [メッセージフィードバックを送信](/ja/api-reference/feedback/submit-message-feedback) で返信を `like` または `dislike` として評価し、任意でコメントを付けられます。`rating` に `null` を渡すと、それまでの評価を取り消せます。[アプリのフィードバック一覧を取得](/ja/api-reference/feedback/list-app-feedbacks) は、エンドユーザーと管理者の両方から寄せられたフィードバックをすべて返します。

## アノテーションの管理

アノテーションは質問に固定の回答を紐づけ、一致した場合はアプリが新たな回答を生成せずそのまま返します。管理には [アノテーションを作成](/ja/api-reference/annotations/create-annotation)、[アノテーションリストを取得](/ja/api-reference/annotations/list-annotations)、[アノテーションを更新](/ja/api-reference/annotations/update-annotation)、[アノテーションを削除](/ja/api-reference/annotations/delete-annotation) を使用します。

[アノテーション返信を設定](/ja/api-reference/annotations/configure-annotation-reply) は、アノテーションによる自動応答のオン・オフを切り替えます。処理は非同期で実行されるため、返されたジョブ ID を使って [アノテーション返信の初期設定タスクステータスを取得](/ja/api-reference/annotations/get-annotation-reply-job-status) を呼び出し、完了を確認します。
