テキストジェネレーターアプリは `completion` モードで動作します。呼び出しごとに入力値を送信し、1 件の応答を返します。呼び出し間で会話状態は保持されません。翻訳、文章作成、要約など、リクエストごとに完結する単発のタスクに適しています。

## テキストの生成

- **[完了メッセージを送信](/ja/api-reference/completion-messages/send-completion-message)**：`inputs` の値をアプリのプロンプトテンプレートに反映し、応答を生成します。ブロッキングモードでは生成の完了後に応答をまとめて返します。ストリーミングモードでは `message` イベントを `message_end` まで連続で受信できます。自動再生が有効な場合は `tts_message` イベントも届きます。
- **[生成を停止](/ja/api-reference/completion-messages/stop-completion-message-generation)**：ストリーミング中の応答を完了前に中断します。

## ファイルの操作

- **[ファイルをアップロード](/ja/api-reference/files/upload-file)**：画像、ドキュメント、音声、動画ファイルをアップロードし、[完了メッセージを送信](/ja/api-reference/completion-messages/send-completion-message) から参照できるようにします。ファイルはアップロードしたエンドユーザーのみが利用できます。
- **[ファイルをダウンロード](/ja/api-reference/files/download-file)**：アップロード済みファイルをプレビューまたはダウンロードします。対象はアプリ内のメッセージに紐づくファイルに限られます。
- **[エンドユーザー取得](/ja/api-reference/end-users/get-end-user-info)**：エンドユーザー ID（ファイルアップロードのレスポンスにある `created_by` など）から、そのユーザーの詳細を取得します。

## 音声とテキストの相互変換

- **[音声をテキストに変換](/ja/api-reference/audio/convert-audio-to-text)**：アップロードされた音声ファイル（MP3、M4A、WAV、AMR、MPGA のいずれか、最大 30 MB）を文字起こしします。エンドユーザーはタイピングの代わりに音声で入力できます。
- **[テキストを音声に変換](/ja/api-reference/audio/convert-text-to-audio)**：生成された応答を音声として合成します。

## アプリ情報と設定の取得

- **[アプリケーションの基本情報を取得](/ja/api-reference/applications/get-app-info)**：アプリの名前、説明、タグ、モードを返します。
- **[アプリケーションのパラメータ情報を取得](/ja/api-reference/applications/get-app-parameters)**：入力フォームの設定（機能スイッチ、パラメータの名前・型・デフォルト値）を返します。クライアント UI の構築に役立ちます。
- **[アプリケーションのメタ情報を取得](/ja/api-reference/applications/get-app-meta)**：ツールアイコンなどの設定メタデータを返します。
- **[アプリの WebApp 設定を取得](/ja/api-reference/applications/get-app-webapp-settings)**：WebApp のサイト設定、テーマ、カスタマイズ項目を返します。

## フィードバックの収集

- **[メッセージフィードバックを送信](/ja/api-reference/feedback/submit-message-feedback)**：エンドユーザーが応答を `like` または `dislike` で評価します。任意でコメントも付けられます。
- **[アプリのフィードバック一覧を取得](/ja/api-reference/feedback/list-app-feedbacks)**：エンドユーザーと管理者の両方から寄せられた、アプリ全体のフィードバックをすべて返します。
