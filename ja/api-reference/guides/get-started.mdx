Dify で公開したアプリは、そのまま REST API としても使えます。ナレッジベースも同様です。エンドユーザーが使うのと同じアプリを、API キーを使って自分のバックエンドから呼び出せます。

## API キーの取得

アプリの場合：アプリ内で API キーを作成します。キーはそのアプリ 1 つにのみ有効です。

ナレッジベースの場合：**ナレッジ** 画面を開き、右上の **サービスAPI** をクリックします。ナレッジベースのキーは範囲が広く、キーを作成したアカウントから見えるすべてのナレッジベースにアクセスできます。取り扱いには特に注意してください。詳細は [ナレッジ API](/ja/api-reference/guides/knowledge) ガイドを参照してください。

<Warning>
API はバックエンドからのみ呼び出してください。フロントエンドコードやクライアントアプリに埋め込んだキーは、抽出されて悪用されるおそれがあります。
</Warning>

## 最初のリクエストを送る

すべてのリクエストは、キーを Bearer トークンとして送信します。Dify Cloud のベース URL は `https://api.dify.ai/v1` です。セルフホスト環境では、自分のインスタンスの API ベース URL に置き換えてください。

まずはすべてのアプリタイプで使える [アプリの基本情報を取得](/ja/api-reference/applications/get-app-info) を試してみましょう：

```bash
curl https://api.dify.ai/v1/info \
  -H "Authorization: Bearer $DIFY_API_KEY"
```

次のようなレスポンスが返れば、キーは有効です。どのアプリのキーかも確認できます：

```json
{
  "name": "My Chat App",
  "description": "A helpful assistant",
  "mode": "chat",
  "tags": ["assistant"]
}
```

ナレッジベースの API キーを使う場合、エンドポイントは別系統です。最初のリクエストには [ナレッジベースリストを取得](/ja/api-reference/knowledge-bases/list-knowledge-bases) を試してください。

## アプリタイプごとのエンドポイントを見つける

次に呼び出せるものは、キーの背後にあるアプリタイプで決まります。各アプリタイプのページで API 全体を確認できます：[チャットフロー](/ja/api-reference/guides/chatflow)、[ワークフロー](/ja/api-reference/guides/workflow)、[New Agent](/ja/api-reference/guides/agent)、[ナレッジ](/ja/api-reference/guides/knowledge)、[ChatbotとAgent](/ja/api-reference/guides/chat)、[テキストジェネレーター](/ja/api-reference/guides/completion)。

リファレンスの各エンドポイントページには対象のアプリタイプが記載されているため、任意のエンドポイントから調べることもできます。
