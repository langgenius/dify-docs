テキストジェネレーターは `completion` モードで動作します。呼び出しごとに入力値を送信し、1 件の応答を返します。呼び出し間で会話状態は保持されず、各リクエストは独立しています。翻訳、文章作成、要約など、単発のタスクに適しています。

## テキストの生成

[完了メッセージを送信](/ja/api-reference/completion-messages/send-completion-message) は、`inputs` の値をアプリのプロンプトテンプレートに反映し、応答を生成します。`response_mode` を `streaming` にすると、`message` イベントを `message_end` まで連続で受信できます。自動再生が有効な場合は、`tts_message` イベントも届きます。`blocking` にすると、応答を 1 回で受信します。

[生成を停止](/ja/api-reference/completion-messages/stop-completion-message-generation) は、`task_id` を指定して実行中のタスクを取り消します。ただし `streaming` モードでのみ有効です。`blocking` モードでは、レスポンスが返る時点ですでに実行が完了しているため取り消せません。

## ファイルの操作

[ファイルをアップロード](/ja/api-reference/files/upload-file) は、画像、ドキュメント、音声、動画ファイルをアップロードし、[完了メッセージを送信](/ja/api-reference/completion-messages/send-completion-message) から参照できるようにします。アップロードしたエンドユーザー自身のみが利用できます。[ファイルをダウンロード](/ja/api-reference/files/download-file) は、そのファイルをプレビューまたはダウンロードします。対象はアプリ内のメッセージに紐づくファイルに限られます。

アップロードファイルのレスポンスの `created_by` のように、レスポンスにはエンドユーザー ID のみが含まれることがあります。その場合は、[エンドユーザー取得](/ja/api-reference/end-users/get-end-user-info) で詳細情報を取得できます。

## 音声とテキストの相互変換

[音声をテキストに変換](/ja/api-reference/audio/convert-audio-to-text) は、アップロードされた音声ファイル（MP3、M4A、WAV、AMR、MPGA のいずれか、最大 30 MB）をテキストに変換します。エンドユーザーは入力をテキストではなく音声で行えます。[テキストを音声に変換](/ja/api-reference/audio/convert-text-to-audio) は、生成された応答を音声として合成します。

## アプリ情報と設定の取得

[アプリケーションの基本情報を取得](/ja/api-reference/applications/get-app-info) は、アプリの名前、説明、タグ、モードを返します。[アプリケーションのパラメータ情報を取得](/ja/api-reference/applications/get-app-parameters) は、機能スイッチや入力パラメータの名前・型・デフォルト値といった入力フォームの設定を返します。クライアント UI の構築に役立ちます。

[アプリケーションのメタ情報を取得](/ja/api-reference/applications/get-app-meta) は、ツールアイコンなどの設定メタデータを返します。[アプリの WebApp 設定を取得](/ja/api-reference/applications/get-app-webapp-settings) は、WebApp のサイト設定、テーマ、カスタマイズ項目を返します。

## フィードバックの収集

エンドユーザーは [メッセージフィードバックを送信](/ja/api-reference/feedback/submit-message-feedback) で応答を `like` または `dislike` として評価し、任意でコメントを付けられます。`rating` に `null` を渡すと、それまでの評価を取り消せます。[アプリのフィードバック一覧を取得](/ja/api-reference/feedback/list-app-feedbacks) は、エンドユーザーと管理者の両方から寄せられたフィードバックをすべて返します。
