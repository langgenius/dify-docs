## デプロイ前の準備

マシンが以下の最小システム要件を満たしているか確認してください。

### ハードウェア

- CPU >= 2 コア
- RAM >= 4 GiB

### ソフトウェア

| オペレーティングシステム | 必要なソフトウェア | 備考 |
| :--- | :--- | :--- |
| macOS 10.14 以降 | Docker Desktop | Docker 仮想マシンに 2 vCPU 以上、8 GiB 以上のメモリを割り当てます。<br></br><br></br>[Mac 用 Docker Desktop のインストール](https://docs.docker.com/desktop/mac/install/) を参照してください。 |
| Linux | Docker 19.03+ / Docker Compose 1.28+ | 手順は以下を参照してください。<br></br><br></br>[Docker Engine のインストール](https://docs.docker.com/engine/install/) / [Docker Compose のインストール](https://docs.docker.com/compose/install/) |
| Windows (WSL 2) | Docker Desktop | ソースコードとデータは Linux ファイルシステムに保存します。<br></br><br></br>[Windows 用 Docker Desktop のインストール](https://docs.docker.com/desktop/windows/install/#wsl-2-backend) を参照してください。 |

## デプロイと起動

<Steps>
  <Step title="Dify をクローン">
    Dify のソースコードをローカルマシンにクローンします。

    ```bash
    git clone --branch "$(curl -s https://api.github.com/repos/langgenius/dify/releases/latest | jq -r .tag_name)" https://github.com/langgenius/dify.git
    ```
  </Step>
  <Step title="Dify を起動">

    1. Dify ソースコード内の `docker` ディレクトリに移動します：

        ```bash
        cd dify/docker
        ```

    2. コンテナを起動します：

        <CodeGroup>

          ```bash macOS/Linux/WSL
          ./dify-compose up -d
          ```

          ```powershell Windows
          .\dify-compose.ps1 up -d
          ```
        </CodeGroup>

        <Info>
          初回実行時、`dify-compose` 起動スクリプトが `.env` を作成し、自動生成された `SECRET_KEY` を書き込みます。
        </Info>

        次に、カスタムデプロイが必要か尋ねられます：

        - **`Enter` を押す（または `n` と回答）** と、`.env.default` のデフォルト値でデプロイされます。コンテナはすぐに起動します。

        - **`y` と回答** すると、先にカスタマイズできます。起動スクリプトが終了し、`.env` を編集できます：

            1. 完全な環境変数リストは `.env.example` を、詳細は [環境変数](/ja/self-host/configuration/environments) を参照してください。

            2. 上書きしたい変数を `.env` に追加します。

            3. 準備ができたら起動コマンドを再度実行します。実行時、`.env` のオーバーライドは `.env.default` のデフォルト値とマージされます。

        <Info>
          このプロンプトは初回実行時のみ表示されます。設定を後で変更する場合は [カスタマイズ](#カスタマイズ) を参照してください。
        </Info>

        以下のコンテナが起動します：

        - 5 つのコアサービス：`api`、`worker`、`worker_beat`、`web`、`plugin_daemon`
        - 6 つの依存コンポーネント：`weaviate`、`db_postgres`、`redis`、`nginx`、`ssrf_proxy`、`sandbox`

        各コンテナのステータスと起動時間を示す以下のような出力が表示されます：

    3. すべてのコンテナが正常に動作しているか確認します：

## アクセス

1. 管理者初期化ページを開いて、管理者アカウントを設定します：

    ```bash
    # ローカル環境
    http://localhost/install

    # サーバー環境
    http://your_server_ip/install
    ```

2. 管理者アカウントの設定が完了したら、以下のアドレスで Dify にログインします：

    ```bash
    # ローカル環境
    http://localhost  

    # サーバー環境
    http://your_server_ip
    ```

## カスタマイズ

完全な環境変数リストは `.env.example` を参照してください。上書きしたい変数をローカルの `.env` ファイルに追加し、Dify を再起動して変更を適用します：

<CodeGroup>

  ```bash macOS/Linux/WSL
  ./dify-compose down
  ./dify-compose up -d
  ```

  ```powershell Windows
  .\dify-compose.ps1 down
  .\dify-compose.ps1 up -d
  ```

</CodeGroup>

<Tip>
  詳細は [環境変数](/ja/self-host/configuration/environments) を参照してください。
</Tip>

## アップグレード

アップグレード手順はリリースによって異なる場合があります。[Releases](https://github.com/langgenius/dify/releases) ページで提供されている対象バージョンのアップグレードガイドを参照してください。

<Note>

  アップグレード後、`.env.default` と `.env.example` の変更を確認してください。`.env` にはローカルのオーバーライドのみを残します。

</Note>
