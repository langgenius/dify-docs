統合は、Dify を外部世界に接続します。アプリを動かすモデルプロバイダー、Agent やワークフローが呼び出すツール、ナレッジベースを支えるデータソース、構築した内容をトリガーまたは拡張するサービスを提供します。

統合は一度インストールすれば、ワークスペース内のすべてのアプリで使用できます。**統合** からすべてをまとめて閲覧・管理できます。

## 統合の種類

- [**モデルプロバイダー**](/ja/self-host/use-dify/workspace/model-providers)：アプリを動かすモデルです。LLM、埋め込みモデル、リランクモデルなどが含まれます。

- [**ツール**](/ja/self-host/use-dify/workspace/tools)：Agent やワークフローが呼び出せる機能です。組み込みツールプラグイン、MCP サーバー、公開済みワークフロー、OpenAPI サービスなどがあります。

- [**データソース**](/ja/self-host/use-dify/knowledge/knowledge-pipeline/authorize-data-source)：ナレッジパイプラインで構築したナレッジベース向けの外部コンテンツです。Notion ページや Web サイトなどがあります。

- [**トリガー**](/ja/self-host/use-dify/nodes/trigger/plugin-trigger)：外部イベントの発生時にワークフローを自動的に開始します。

- **エージェント戦略**：[Agent ノード](/ja/self-host/use-dify/nodes/agent) がツールの計画と呼び出しに使用する推論戦略です。

- [**カスタム API**](/ja/self-host/use-dify/workspace/api-extension/api-extension)：独自の API を接続し、コンテンツモデレーションや外部データツールをアプリに追加します。

- **拡張**：HTTP Webhook を使用して外部サービスをアプリに統合します。

## 統合のインストール

統合は 3 つのソースから追加できます：

- [**マーケットプレイス**](https://marketplace.dify.ai/)：テストおよびメンテナンスされた、公式およびパートナーの統合です。
- **GitHub**：URL とバージョンを指定して、任意の公開リポジトリからインストールします。
- **ローカルアップロード**：プライベートまたは内部の統合用のカスタム `.zip` パッケージです。

<Tip>
  Dify Enterprise では、管理者がデプロイ全体でインストール元を制限し、公式の統合のみ、公式および認証パートナーの統合、またはマーケットプレイスからのインストールのみに限定できます。詳しくは [営業担当](https://udify.app/chat/QuwcpW1oBNcfeL55) までお問い合わせください。
</Tip>

## 権限と更新

ワークスペースのオーナーと管理者は、統合に対して 2 つのワークスペース全体の権限を設定でき、それぞれ **みんな**、**管理者**、**誰もいない** のいずれかを選べます：

- **統合をインストールおよび管理できるユーザー**：インストール、アップグレード、削除。デフォルトは **みんな** です。
- **統合をデバッグできるユーザー**：デバッグキーへのアクセス。デフォルトは **誰もいない** です。

統合は自動的に更新することもできます。カテゴリ（ツールやモデルプロバイダーなど）ごとに、更新戦略（オフ、パッチバージョンのみ、または常に最新）と、すべての統合に適用するか一部のみに適用するかを選択します。

インストール後、ほとんどの統合には API キーやエンドポイントなどの設定が必要で、これらはワークスペース全体に適用されます。
