Dify では、プラグインを配布する方法を 3 つ用意しています。これらは、事前にどれだけ審査されるか、そしてプラグインが届く対象範囲の広さで異なります。

公開前に、開発とテストを完了し、[プラグイン開発ガイドライン](/ja/develop-plugin/publishing/standards/contributor-covenant-code-of-conduct) を読んでください。

## 方法の選択

<CardGroup cols={3}>
  <Card title="マーケットプレイス" icon="store" href="/ja/develop-plugin/publishing/marketplace-listing/release-to-dify-marketplace">
    公式カタログです。審査済みで信頼でき、すべての Dify ユーザーがワンクリックでインストールできます。
  </Card>
  <Card title="GitHub リポジトリ" icon="github" href="/ja/develop-plugin/publishing/marketplace-listing/release-to-individual-github-repo">
    GitHub 上でセルフホストします。審査はなく、ユーザーはリポジトリの URL からインストールします。
  </Card>
  <Card title="ローカルファイル" icon="file-zipper" href="/ja/develop-plugin/publishing/marketplace-listing/release-by-file">
    直接手渡しする `.difypkg` ファイルです。プライベートまたは社内向けのプラグインに適しています。
  </Card>
</CardGroup>

## 比較

| | <Icon icon="store" /> マーケットプレイス | <Icon icon="github" /> GitHub | <Icon icon="file-zipper" /> ローカルファイル |
| :--- | :--- | :--- | :--- |
| **対象** | すべての Dify ユーザー | リポジトリの URL を知っている人 | ファイルを渡した相手 |
| **審査** | あり: 12 項目の自動チェック + 人による審査 | なし | なし |
| **インストール経路** | マーケットプレイスの UI からワンクリック | GitHub ダイアログからインストール | プラグインページからアップロード |
| **バージョン管理** | バージョンごとに新しい PR、自動化ワークフローも利用可能 | `.difypkg` アセット付きの GitHub リリース | 再パッケージ化して再配布 |
| **適したケース** | 完成度が高く、幅広く役立つプラグイン | オープンソースプロジェクト、URL で共有する社内ツール | プライベートプラグイン、社内テスト、アドホックな配布 |

<Tip>
多くの開発者は、まず **GitHub** に公開して素早く反復し、プラグインが安定してから **マーケットプレイス** に申請します。同じ `.difypkg` が 3 つの方法すべてで使え、異なるのは配布チャネルだけです。
</Tip>

## 選び方の目安

- **幅広く届けて審査も受けたい場合は？** **マーケットプレイス** を使います。
- **リリースとバージョン管理を完全に制御したい場合は？** **GitHub** を使います。
- **プライベートまたは一度きりの用途は？** **ローカルファイル** を使います。
