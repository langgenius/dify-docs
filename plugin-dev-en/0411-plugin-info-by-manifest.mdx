Your manifest is the contract between your plugin and the platform. Use this reference to describe capabilities accurately, avoid packaging errors, and streamline marketplace review.

The Manifest is a YAML-compliant file that defines the most basic information of a **plugin**, including but not limited to the plugin name, author, included tools, models, etc. For the overall architecture of the plugin, please refer to [Basic Concepts of Plugin Development](/plugin-dev-en/0111-getting-started-dify-plugin) and [Developer Cheatsheet](/plugin-dev-en/0131-cheatsheet).

If the format of this file is incorrect, the parsing and packaging process of the plugin will fail.

### Code Example

Below is a simple example of a Manifest file. The meaning and function of each data item will be explained later.

For reference code of other plugins, please refer to the [GitHub code repository](https://github.com/langgenius/dify-official-plugins/blob/main/tools/google/manifest.yaml).

```yaml
version: 0.2.0
type: "plugin"
author: "acme-labs"
name: "sentiment-analyzer"
label:
  en_US: "Sentiment Analyzer"
  zh_Hans: "情绪分析"
description:
  en_US: "Adds sentiment scoring to your customer-support workflow."
  zh_Hans: "为客户支持流程提供情绪分析能力。"
created_at: "2025-02-10T10:05:32Z"
icon: "_assets/icon.svg"
icon_dark: "_assets/icon-dark.svg"
tags:
  - "productivity"
resource:
  memory: 1048576
  permission:
    tool:
      enabled: true
    app:
      enabled: true
    storage:
      enabled: true
      size: 1048576
plugins:
  tools:
    - "provider/sentiment.yaml"
meta:
  version: "0.2.0"
  arch:
    - "amd64"
    - "arm64"
  minimum_dify_version: "1.1.0"
  runner:
    language: "python"
    version: "3.12"
    entrypoint: "main"
privacy: "PRIVACY.md"
repo: "https://github.com/acme-labs/dify-plugin-sentiment"
```

### Structure

<ParamField path="version" type="version" required>
  The version of the plugin.
</ParamField>

<ParamField path="type" type="string" required>
  Manifest classification. For plugin packages this value must be `plugin`.
</ParamField>

<ParamField path="author" type="string" required>
  Plugin author namespace (lowercase letters, numbers, `_`, `-`, max 64 characters).
</ParamField>

<ParamField path="name" type="string" required>
  Plugin slug (lowercase letters, numbers, `_`, `-`, max 128 characters).
</ParamField>

<ParamField path="label" type="object" required>
  Multilingual display name.
</ParamField>

<ParamField path="description" type="object" required>
  Multilingual description shown in the Marketplace.
</ParamField>

<ParamField path="created_at" type="RFC3339" required>
  Creation timestamp. Must not be later than the packaging time.
</ParamField>

<ParamField path="icon" type="string" required>
  Light-theme icon path.
</ParamField>

<ParamField path="icon_dark" type="string">
  Optional dark-theme icon path.
</ParamField>

<ParamField path="tags" type="array[string]">
  Marketplace tags. Allowed values: `search`, `image`, `videos`, `weather`, `finance`, `design`, `travel`, `social`, `news`, `medical`, `productivity`, `education`, `business`, `entertainment`, `utilities`, `agent`, `rag`, `other`.
</ParamField>

<ParamField path="resource" type="object" required>
  Resources to apply for.
  
  <ParamField path="memory" type="int64" required>
    Maximum memory usage, mainly related to AWS Lambda resource application on SaaS, unit in bytes.
  </ParamField>
  
  <ParamField path="permission" type="object">
    Permission application. Omit this object to request no additional capabilities (the default).
    
    <ParamField path="tool" type="object">
      Permission for reverse invocation of tools.
      
      <ParamField path="enabled" type="boolean">
        Whether to enable tool permissions.
      </ParamField>
    </ParamField>
    
    <ParamField path="model" type="object">
      Permission for reverse invocation of models.
      
      <ParamField path="enabled" type="boolean">
        Whether to enable model permissions.
      </ParamField>
      
      <ParamField path="llm" type="boolean">
        Whether to enable large language model permissions.
      </ParamField>
      
      <ParamField path="text_embedding" type="boolean">
        Whether to enable text embedding model permissions.
      </ParamField>
      
      <ParamField path="rerank" type="boolean">
        Whether to enable rerank model permissions.
      </ParamField>
      
      <ParamField path="tts" type="boolean">
        Whether to enable text-to-speech model permissions.
      </ParamField>
      
      <ParamField path="speech2text" type="boolean">
        Whether to enable speech-to-text model permissions.
      </ParamField>
      
      <ParamField path="moderation" type="boolean">
        Whether to enable content moderation model permissions.
      </ParamField>
    </ParamField>
    
    <ParamField path="node" type="object">
      Permission for reverse invocation of nodes.
      
      <ParamField path="enabled" type="boolean">
        Whether to enable node permissions.
      </ParamField>
    </ParamField>
    
    <ParamField path="endpoint" type="object">
      Permission to register `endpoint`.
      
      <ParamField path="enabled" type="boolean">
        Whether to enable endpoint permissions.
      </ParamField>
    </ParamField>
    
    <ParamField path="app" type="object">
      Permission for reverse invocation of `app`.
      
      <ParamField path="enabled" type="boolean">
        Whether to enable app permissions.
      </ParamField>
    </ParamField>
    
    <ParamField path="storage" type="object">
      Permission to apply for persistent storage.
      
      <ParamField path="enabled" type="boolean">
        Whether to enable storage permissions.
      </ParamField>
      
      <ParamField path="size" type="uint64">
        Maximum allowed persistent memory size, unit in bytes. Required when storage is enabled. Acceptable values range from `1024` (1&nbsp;KB) to `1073741824` (1&nbsp;GB).
      </ParamField>
    </ParamField>
  </ParamField>
</ParamField>

<ParamField path="plugins" type="object" required>
  A list of `yaml` files for the specific capabilities extended by the plugin. Absolute path within the plugin package. For example, if you need to extend a model, you need to define a file similar to `openai.yaml`, fill in the file path here, and the file at this path must actually exist, otherwise packaging will fail.

  <Warning>
    Having no extensions is not allowed.
  </Warning>

  <ParamField path="tools" type="array[string]">
    Plugin extension for [Tool](/plugin-dev-en/0222-tool-plugin) providers.
  </ParamField>
  
  <ParamField path="models" type="array[string]">
    Plugin extension for [Model](/plugin-dev-en/0222-creating-new-model-provider) providers.
  </ParamField>
  
  <ParamField path="endpoints" type="array[string]">
    Plugin extension for [Endpoints](/plugin-dev-en/0432-develop-a-slack-bot-plugin) providers.
  </ParamField>
  
  <ParamField path="agent_strategies" type="array[string]">
    Plugin extension for [Agent Strategy](/plugin-dev-en/9433-agent-strategy-plugin) providers.
  </ParamField>

  <ParamField path="datasources" type="array[string]">
    Plugin extension for datasource providers.
  </ParamField>

  <Warning>
    You cannot declare model, datasource, or agent-strategy providers alongside any other provider type.
  </Warning>

  <Warning>
    Currently only one provider file per type is supported.
  </Warning>
</ParamField>

<ParamField path="meta" type="object" required>
  Metadata for runtime packaging and compatibility.
  
  <ParamField path="version" type="version" required>
    Manifest schema version (use semantic versioning, e.g., `0.2.0`).
  </ParamField>
  
  <ParamField path="arch" type="array[string]" required>
    Supported CPU architectures. The CLI currently validates `amd64` and `arm64`.
  </ParamField>
  
  <ParamField path="runner" type="object" required>
    Runtime configuration.
    
    <ParamField path="language" type="string" required>
      Plugin runtime language. Validation presently allows only `python`.
    </ParamField>
    
    <ParamField path="version" type="string" required>
      Language runtime version. Set this to the interpreter version your plugin expects (the daemon currently provisions Python `3.12`).
    </ParamField>
    
    <ParamField path="entrypoint" type="string" required>
      Module name passed to `python -m`. Defaults to `main` in generated templates.
    </ParamField>
  </ParamField>

  <ParamField path="minimum_dify_version" type="string">
    Optional semantic version that guards installation on older Dify releases.
  </ParamField>
</ParamField>

<ParamField path="privacy" type="string">
  Specifies the relative path or URL of the plugin's privacy policy file, e.g., `"./privacy.md"` or `"https://your-web/privacy"`. If you plan to list the plugin on the Dify Marketplace, **this field is required** to provide clear user data usage and privacy statements. For detailed filling guidelines, please refer to [Plugin Privacy Data Protection Guidelines](/plugin-dev-en/0312-privacy-protection-guidelines).
</ParamField>

<ParamField path="repo" type="string">
  Optional URL pointing to the public source repository for your plugin.
</ParamField>
