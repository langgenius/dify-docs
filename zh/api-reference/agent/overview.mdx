New Agent 应用运行在 `agent` 模式下，遵循对话消息流程：发送消息、流式接收回复，并使用与其他对话型应用相同的接口管理会话。模型会自主推理并调用工具，但相关过程不会作为流事件暴露，流中只包含普通的 `message` 事件。

<Note>
New Agent 应用类型目前处于 Beta 阶段。[发送对话消息](/zh/api-reference/chat-messages/send-chat-message) 暂不支持为 New Agent 应用传入文件。
</Note>

## 发送消息并接收流式回复

[发送对话消息](/zh/api-reference/chat-messages/send-chat-message) 向 New Agent 应用发送一次请求。New Agent 应用仅支持 `streaming`；使用 `blocking` 会返回 400 `bad_request` 错误。流中的 `message_end` 事件在 `metadata` 中携带 `usage`（命中标注回复时还包含 `annotation_reply`），但不会包含 `retriever_resources`。

如需在回复完成前中断流式输出，可调用 [停止响应](/zh/api-reference/chat-messages/stop-chat-message-generation)。

## 管理会话

[获取会话列表](/zh/api-reference/conversations/list-conversations) 按最近活跃时间返回当前用户的会话列表。[获取会话历史消息](/zh/api-reference/conversations/list-conversation-messages) 以滚动加载的方式返回某个会话的历史消息，适合聊天界面分页显示；由于模型的推理过程不会对外暴露，每条消息的 `agent_thoughts` 数组始终为空。

## 上传文件

[上传文件](/zh/api-reference/files/upload-file) 用于上传图片、文档、音频或视频文件，仅限上传者本人使用。New Agent 应用尚不支持文件输入：调用 [发送对话消息](/zh/api-reference/chat-messages/send-chat-message) 时，系统会接受传入的文件引用，但不会使用。

## 获取应用信息与配置

[获取应用基本信息](/zh/api-reference/applications/get-app-info) 返回应用的名称、描述、标签和模式。[获取应用参数](/zh/api-reference/applications/get-app-parameters) 返回输入表单配置：功能开关、输入参数名称、类型及默认值，可用于构建客户端界面。

[获取应用元数据](/zh/api-reference/applications/get-app-meta) 返回工具图标等配置元数据。
