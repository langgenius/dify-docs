[New Agent 应用](/zh/self-host/use-dify/build/new-agent/overview) 运行在 `agent` 模式下：发送消息、流式接收生成中的回复，并通过会话保留多轮上下文。

模型会自主推理并调用工具，这些过程也会出现在事件流中：回复以普通的 `message` 文本片段返回，每个推理步骤和工具调用则以 `agent_thought` 事件同步返回。

<Note>
New Agent 应用类型目前处于 Beta 阶段。
</Note>

## 发送消息并接收流式回复

- **[发送对话消息](/zh/api-reference/chat-messages/send-chat-message)**：向 New Agent 应用发送用户提问。仅支持流式返回，使用阻塞式返回会收到 400 `bad_request` 错误。

    收尾的 `message_end` 事件会报告 token 用量，但不会包含知识检索引用（`retriever_resources`）。

- **[停止响应](/zh/api-reference/chat-messages/stop-chat-message-generation)**：在回复完成前中断流式输出。

## 管理会话

- **[获取会话列表](/zh/api-reference/conversations/list-conversations)**：按最近活跃时间排序，返回当前用户的会话。
- **[获取会话历史消息](/zh/api-reference/conversations/list-conversation-messages)**：返回某个会话的历史消息，适合滚动加载的聊天界面。历史记录同样不包含推理过程，每条消息的 `agent_thoughts` 数组始终为空。

## 上传文件

- **[上传文件](/zh/api-reference/files/upload-file)**：上传图片、文档、音频或视频文件，仅限上传的终端用户使用。New Agent 应用尚不支持文件输入：调用 [发送对话消息](/zh/api-reference/chat-messages/send-chat-message) 时，系统会接受传入的文件引用，但不会处理文件内容。

## 获取应用信息与配置

- **[获取应用基本信息](/zh/api-reference/applications/get-app-info)**：返回应用的名称、描述、标签和模式。
- **[获取应用参数](/zh/api-reference/applications/get-app-parameters)**：返回输入表单配置（功能开关、参数名称、类型及默认值），可用于构建客户端界面。
- **[获取应用元数据](/zh/api-reference/applications/get-app-meta)**：返回工具图标等配置元数据。
