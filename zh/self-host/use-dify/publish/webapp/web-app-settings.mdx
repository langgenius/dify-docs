Web App 设置决定你发布的应用在终端用户面前的外观与行为。每个 Dify 应用都会自动生成一个适应不同设备和屏幕尺寸的 Web 界面。

## 设置工作原理

Web 应用始终反映应用的当前配置。修改设置并点击 **发布** 后，更改会立即应用到用户正在访问的线上 Web 应用。

<Info>
Web 应用默认公开。任何拥有 URL 的人都可访问，适用于演示、公共工具和面向客户的应用。
</Info>

### 基本品牌元素

**应用图标和名称**
- 你的图标会出现在浏览器标签页中，以及用户收藏你的应用时
- 选择一个清晰、易于识别的名称来解释你的应用功能
- 图标可以是图片或表情符号，选择符合你应用个性的

**描述与文案**
- 编写简洁的描述，帮助用户理解应用的用途
- 此文本会出现在应用的着陆页和搜索结果中
- 为获得最佳效果，请保持在 160 个字符以内

**视觉一致性**
- 选择与你的品牌一致的颜色
- 在选择明亮或暗色主题时考虑你的受众
- 在不同设备上测试你的应用，确保各处显示正常

## 访问控制

Web 应用默认公开。任何拥有 URL 的人都可打开，适用于演示、公共工具和面向客户的应用。

<Tip>
  在 Dify 企业版中，你可以通过身份验证、成员级访问控制和外部用户管理来限制谁能打开 Web 应用。[联系销售](https://dify.ai/pricing) 了解更多。
</Tip>

**隐私与法律信息**：
- 添加版权信息和隐私政策链接
- 配置数据处理偏好
- 设置服务条款以符合合规要求

## 功能继承

你的 Web 应用会自动包含你在应用程序配置中启用的功能：

**交互功能**：
- 对话开启器和建议的后续操作
- 对话前表单用于上下文收集
- 语音输入和语音转文字能力
- 来源引用与归属和参考链接
- 反馈收集和评分系统

**功能性**：
- 所有工作流步骤和 AI 模型配置
- 知识库集成和工具连接
- 自定义提示词和响应格式
- 速率限制和使用控制

<Warning>
在你的应用配置中禁用功能会立即从发布的 Web 应用中移除它们。
</Warning>

## 应用类型和行为

你的 Web 应用会根据你的应用程序类型自动适应其界面：

<Tabs>
  <Tab title="对话应用程序">

    **功能**：持久会话、对话管理、实时响应
    **最适合**：客户支持、咨询工具、交互式助手
  </Tab>
  
  <Tab title="工作流应用程序">

    **功能**：单次运行、批处理、结果保存
    **最适合**：内容生成、数据处理、分析工具
  </Tab>
