工作空间是你团队在 Dify 中的完整 AI 环境。它包含并隔离你的组织所需的一切：应用程序、知识库、团队成员、模型配置和集成。

## 工作空间概念模型

Dify 中的每个资源都属于一个工作空间。当你创建应用时，它会继承工作空间的模型配置。当你添加团队成员时，他们会根据其角色获得对工作空间资源的访问权限。当你配置模型或安装集成时，它们对整个工作空间都可用。

```
🏢 你的组织
└── 📁 工作空间
    ├── 🤖 应用程序（聊天机器人、工作流、Agent）
    ├── 📊 知识库（文档、嵌入向量）
    ├── 👥 团队成员（角色和权限） 
    ├── 🧠 模型供应商（API 密钥、配置）
    └── 🔧 集成（工具、触发器及其他扩展）
```

这种以工作空间为核心的设计意味着你的资源与其他组织完全隔离，团队成员只能访问他们被允许查看的内容，你只需为整个工作空间配置一次模型。

## 工作空间创建

安装时会创建一个工作空间，使用你在初始化阶段设置的管理员邮箱和密码。

<Tip>
  在 Dify 企业版中，你可以运行多个工作空间，在不同法律实体、监管环境或客户项目之间实现完全隔离。如需了解更多，请 [联系销售](https://udify.app/chat/QuwcpW1oBNcfeL55)。
</Tip>

## 资源如何连接

你构建的应用程序可以使用工作空间中配置的任何模型供应商，访问所有工作空间知识库，并使用已安装的集成。团队成员根据其工作空间权限查看应用程序。

工作空间角色决定对所有资源的访问权限：
- **所有者** 控制模型供应商和工作空间设置
- **管理员** 管理团队成员并配置模型和集成
- **编辑者** 构建应用程序并管理知识库
- **成员** 使用已发布的应用程序

## 工作空间导航

Dify 围绕工作空间概念组织一切。主导航提供 Studio（你的应用）、知识库、集成（模型供应商、工具、触发器及其他扩展）和市场的入口。设置中包含成员、语言等工作空间偏好。
