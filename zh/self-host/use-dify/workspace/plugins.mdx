集成将 Dify 连接到外部世界。它们提供驱动应用的模型供应商、Agent 与工作流所调用的工具、知识库背后的数据来源，以及用于触发或扩展应用的各种服务。

集成只需安装一次，工作空间中的每个应用都可使用。在 **集成** 中即可浏览和管理它们。

## 集成类型

- [**模型供应商**](/zh/self-host/use-dify/workspace/model-providers)：驱动应用的各类模型，涵盖 LLM、嵌入模型与重排序模型。

- [**工具**](/zh/self-host/use-dify/workspace/tools)：Agent 和工作流可调用的能力，包括内置工具插件、MCP 服务器、已发布的工作流以及 OpenAPI 服务。

- [**数据来源**](/zh/self-host/use-dify/knowledge/knowledge-pipeline/authorize-data-source)：通过知识流水线构建的知识库所用的外部内容，如 Notion 页面或网站。

- [**触发器**](/zh/self-host/use-dify/nodes/trigger/plugin-trigger)：在外部事件发生时自动启动工作流。

- **Agent 策略**：[Agent 节点](/zh/self-host/use-dify/nodes/agent) 用于规划和调用工具的推理策略。

- [**自定义端点**](/zh/self-host/use-dify/workspace/api-extension/api-extension)：连接你自己的 API，为应用添加内容审核或外部数据工具。

- **扩展**：通过 HTTP Webhook 将外部服务集成到应用中。

## 安装集成

可从三种来源添加集成：

- [**市场**](https://marketplace.dify.ai/)：官方和合作伙伴提供的集成，已经过测试和维护。
- **GitHub**：通过 URL 和版本从任意公共仓库安装。
- **本地上传**：用于私有或内部集成的自定义 `.zip` 包。

<Tip>
  在 Dify 企业版中，管理员可在整个部署范围内限制安装来源，仅允许官方集成、官方及认证合作伙伴的集成，或仅允许从市场安装。如需了解更多，请 [联系销售](https://udify.app/chat/QuwcpW1oBNcfeL55)。
</Tip>

## 权限与更新

工作空间所有者和管理员可为集成设置两项工作空间级权限，每项均可设为 **所有人**、**管理员** 或 **无人**：

- **谁可以安装和管理集成**：安装、升级和移除。默认为 **所有人**。
- **谁可以调试集成**：访问调试密钥。默认为 **无人**。

你还可以配置集成自动更新。可按类别（如工具或模型供应商）选择更新策略（关闭、仅修订版本或始终最新），并指定应用于全部集成还是部分集成。

安装后，大多数集成需要配置 API 密钥或端点等信息，这些设置在整个工作空间生效。
