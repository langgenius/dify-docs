## 部署前准备

确保机器满足以下最低系统要求。

### 硬件

- CPU >= 2 Core
- RAM >= 4 GiB

### 软件

| 操作系统 | 所需软件 | 说明 |
| :--- | :--- | :--- |
| macOS 10.14 或更高版本 | Docker Desktop | Docker 虚拟机至少配置 2 个虚拟 CPU 和 8 GiB 内存。<br></br><br></br>安装说明详见 [Mac 版 Docker Desktop 安装指南](https://docs.docker.com/desktop/mac/install/)。 |
| Linux 发行版 | Docker 19.03+ / Docker Compose 1.28+ | 安装说明详见 [Docker 引擎安装指南](https://docs.docker.com/engine/install/) 和 [Docker Compose 安装指南](https://docs.docker.com/compose/install/)。 |
| Windows（启用 WSL 2） | Docker Desktop | 源代码和绑定到 Linux 容器的数据建议存储在 Linux 文件系统中，而非 Windows。<br></br><br></br>安装说明详见 [Windows 版 Docker Desktop 安装指南](https://docs.docker.com/desktop/windows/install/#wsl-2-backend)。 |


## 部署并启动

<Steps>
  <Step title="克隆 Dify">
    克隆 Dify 源代码到本地机器。

    ```bash
    git clone --branch "$(curl -s https://api.github.com/repos/langgenius/dify/releases/latest | jq -r .tag_name)" https://github.com/langgenius/dify.git
    ```
  </Step>
  <Step title="启动 Dify">

    1. 进入 Dify 源代码中的 `docker` 目录：

        ```bash
        cd dify/docker
        ```

    2. 启动容器：

        <CodeGroup>

          ```bash macOS/Linux/WSL
          ./dify-compose up -d
          ```

          ```powershell Windows
          .\dify-compose.ps1 up -d
          ```
        </CodeGroup>

        <Info>
          首次运行时，`dify-compose` 启动脚本创建 `.env` 并写入自动生成的 `SECRET_KEY`。
        </Info>

        脚本将询问是否需要自定义部署：

        - **按 `Enter`（或回答 `n`）**：使用 `.env.default` 中的默认值部署。容器立即启动。

        - **回答 `y`**：先进行自定义。启动脚本将退出，以便你编辑 `.env`：

            1. 参考 `.env.example` 了解完整环境变量列表。详细说明见 [环境变量](/zh/self-host/configuration/environments)。

            2. 将需要覆盖的变量添加到 `.env`。

            3. 再次运行启动命令。运行时，`.env` 中的覆盖项将与 `.env.default` 中的默认值合并。

        <Info>
          该询问仅在首次运行时出现。后续如需修改配置，详见 [自定义](#自定义)。
        </Info>

        启动以下容器：

        - 5 个核心服务：`api`、`worker`、`worker_beat`、`web`、`plugin_daemon`
        - 6 个依赖组件：`weaviate`、`db_postgres`、`redis`、`nginx`、`ssrf_proxy`、`sandbox`

        应看到类似以下的输出，显示每个容器的状态和启动时间：

            ```bash

            ✔ Container docker-ssrf_proxy-1      Started                                                                 0.4s 
            ✔ Container docker-weaviate-1        Started                                                                 0.3s 
            ✔ Container docker-worker_beat-1     Started                                                                 3.2s 
            ✔ Container docker-api-1             Started                                                                 3.2s

            ✔ Container docker-plugin_daemon-1   Started                                                                 3.2s 
            ✔ Container docker-nginx-1           Started                                                                 3.4s 
            ```

    3. 验证所有容器是否成功运行：

        <CodeGroup>

          ```bash macOS/Linux/WSL
          ./dify-compose ps
          ```

          ```powershell Windows
          .\dify-compose.ps1 ps
          ```

        </CodeGroup>

        应看到类似以下的输出，每个容器的状态应为 `Up` 或 `healthy`：


## 访问

1. 打开管理员初始化页面，设置管理员账户：

    ```bash
    # 本地环境
    http://localhost/install

    # 服务器环境
    http://your_server_ip/install
    ```

2. 完成管理员账户设置后，在以下地址登录 Dify：

    ```bash
    # 本地环境
    http://localhost  

    # 服务器环境
    http://your_server_ip
    ```

## 自定义

参考 `.env.example` 查看完整环境变量列表。将需要覆盖的变量添加到本地 `.env` 文件，然后重启 Dify 以应用更改：

<CodeGroup>

  ```bash macOS/Linux/WSL
  ./dify-compose down
  ./dify-compose up -d
  ```

  ```powershell Windows
  .\dify-compose.ps1 down
  .\dify-compose.ps1 up -d
  ```

</CodeGroup>

<Tip>
  更多信息，详见 [环境变量](/zh/self-host/configuration/environments)。
</Tip>

## 升级

不同版本的升级步骤可能有所不同。详见 [Releases](https://github.com/langgenius/dify/releases) 页面中提供的目标版本升级指南。

<Note>

  升级后，查看 `.env.default` 和 `.env.example` 的变更。`.env` 仅保留本地覆盖项。

</Note>
