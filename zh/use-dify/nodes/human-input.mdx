人工输入节点会在关键点暂停工作流，在继续执行之前请求人工输入。

当执行到达此节点时，会通过特定渠道发送可自定义的请求表单。接收者可以提供输入、审核数据，并从预定义的操作中选择（如 `Approve` 或 `Regenerate`），以决定工作流如何继续。

通过在关键位置直接嵌入人工输入，您可以**在自动化效率与人工监督之间取得平衡**。

## 配置

配置以下内容以定义节点如何请求和处理人工输入：

- **发送方式**：请求表单如何发送给接收者。

- **表单内容**：接收者将看到什么信息以及他们可以输入什么。

- **用户操作**：接收者可以采取哪些操作，以及工作流如何相应地继续。

- **超时策略**：如果在设定时间内没有人响应会发生什么。

### 发送方式

选择发送请求表单的渠道。目前可用的方式：

- **Web 应用**：直接在 Web 应用界面中显示请求表单，供当前用户响应。

- **电子邮件**：直接向一个或多个接收者发送包含请求表单链接的电子邮件。

<Note>
    无论发送方式如何，请求在收到第一个响应后即关闭。
</Note>

### 表单内容

自定义请求表单中显示的内容：

- 使用 **Markdown** 语法格式化和组织内容

- 引用可用的**变量**以显示动态数据

    <Tip>
        如果您引用推理模型的 `text` 输出变量，表单将显示模型的思考过程以及最终答案。
    
        如果只想显示答案，请为相应的 LLM 节点开启**启用推理标签分离**。
    </Tip>

- 添加交互式**输入字段**，供接收者提供输入或进行编辑

    <Info>
        输入字段可以预填充变量或静态内容，接收者可以对其进行编辑。
    
        每个输入字段都会成为输出变量，供下游节点使用。
    </Info>

### 用户操作

定义接收者可以点击的决策按钮。

每个按钮会从人工输入节点创建相应的分支，根据所做的决策路由执行。

例如，`Post` 分支可能会通向触发内容发布的节点，而 `Regenerate` 分支可能会循环回到 LLM 节点以修改内容。

<Tip>
    使用预设的按钮样式来从视觉上区分不同操作。
    
    例如，对于 `Approve` 等关键操作使用醒目的样式，对于次要选项使用更柔和的样式。
</Tip>

### 超时策略

设置超时策略以确定在给定时间内没有人响应时工作流如何继续。

- **超时时间**：超时前等待多长时间（以小时或天为单位）。

- **超时路径**：在超时时间内没有人响应时要遵循的分支。它可以通向通知或默认决策。
