当文件直接传递给 LLM 时，处理效果受限于模型自身的多模态能力。虽然[文档提取](/zh/use-dify/nodes/doc-extractor)节点可以将部分文件类型转换为文本供 LLM 处理，但其支持的格式仍然有限。

这就是上传文件到沙盒节点的用武之地。文件上传至沙盒后，下游的 [Agent 节点](/zh/use-dify/nodes/agent#沙盒运行时)（启用 **Agent 模式**）可以通过命令行在沙盒中按需安装工具和运行脚本，将文件转换为 LLM 可处理的格式。

<Note>
    处理效果取决于模型的推理能力。
</Note>

你可以在**产物**中找到上传的文件。它们遵循与其他产物相同的[生命周期规则](/zh/use-dify/build/file-system#产物生命周期)。

**使用方法：**

1. 在上传文件到沙盒节点中，选择一个文件变量（例如来自用户输入的变量）。

2. 在下游 Agent 节点的提示词中，引用 `sandbox_path` 和 `file_name` 变量，以便模型定位和处理文件。例如：

    ```bash wrap
    # 系统指令
    你是一名数据分析师。分析上传的数据文件并提供见解。

    # 用户消息
    分析 {{Upload File to Sandbox.file_name}} 中的数据，文件位于 {{Upload File to Sandbox.sandbox_path}}。
    ```

    <Info>
        `file_name` 变量包含文件扩展名（例如 `data.csv`）。
    </Info>
