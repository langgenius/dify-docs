创建新工作流时，你需要选择一个开始节点，决定工作流是等待有人运行它，还是自行运行。

## 用户输入

当需要由终端用户（或 API 调用）启动工作流并提供所需信息时，使用 [用户输入](/zh/cloud/use-dify/nodes/user-input)。你可以定义文本、数字、文件等输入字段，下游节点将引用这些值。

只有以用户输入启动的工作流才能发布为 Web 应用、MCP 服务器、后端服务 API，或在其他 Dify 应用中作为工具复用。

<Info>
对话流始终以用户输入开始。只有工作流可以使用触发器。
</Info>

## 触发器

当工作流需要自动运行而无需手动启动时，使用 [触发器](/zh/cloud/use-dify/nodes/trigger/overview)。有三种类型：

- **[定时触发器](/zh/cloud/use-dify/nodes/trigger/schedule-trigger)**：按指定时间或间隔运行。
- **[集成触发器](/zh/cloud/use-dify/nodes/trigger/plugin-trigger)**：通过触发器集成的事件订阅，在外部系统发生事件时运行。
- **[Webhook 触发器](/zh/cloud/use-dify/nodes/trigger/webhook-trigger)**：当工作流收到外部系统的 HTTP 请求时运行。

## 切换开始节点

用户输入和触发器在同一画布上互斥。要切换，右键点击当前开始节点并选择 **更改节点**，或删除后重新添加。
