你构建的每个 AI 应用都运行在模型之上。Dify Cloud 内置了热门模型，开箱即用，当你需要更多时，也可连接自己的供应商账户。

模型在整个工作空间内共享，因此团队所有人使用的都是同一批模型。

<Warning>
只有工作空间所有者和管理员可以管理供应商。你添加的任何密钥都在整个工作空间内生效，并通过你自己的供应商账户计费。
</Warning>

## 使用 Dify Cloud 内置的模型

创建工作空间后，你即可使用 Dify Cloud 内置的一组热门模型，无需任何设置。每次 AI 回复都会消耗计划中的 **AI 消息额度**，因此你可以立即开始构建和测试。

一次 AI 回复即一次模型调用（一次输入和一次输出），无论消耗多少 token，都计为一次回复。一次回复消耗的额度取决于模型，模型越大消耗越多。各模型的具体费率详见 [Dify 定价页面](https://dify.ai/pricing) 的 FAQ。

## 在内置供应商上使用你自己的账户

当你需要更高的速率限制，或希望通过自己的供应商账户计费时，可在内置供应商上添加自己的 API 密钥。

1. 前往 **集成** > **模型供应商**，找到目标供应商。

2. 点击其卡片上的 **设置**，然后填入 API 密钥及其他必要信息。Dify 会先验证密钥，再将该供应商投入使用。

你的密钥和 AI 消息额度可同时存在。在供应商卡片上设置 **使用优先级**，即可控制 Dify 优先使用哪一个并回退到另一个，这样在额度用尽后，你添加的密钥便可自动接管。

## 连接 Dify Cloud 未内置的供应商

若要使用未内置的供应商，需先安装它并通过你自己的账户运行。AI 消息额度不适用于这类供应商。

1. 在 **集成** > **模型供应商** 中浏览 **安装模型供应商** 区域，或打开 [Marketplace](https://marketplace.dify.ai/) 查看完整列表。

2. 安装该供应商，点击 **设置**，填入 API 密钥及其他必要信息。

## 添加自定义模型

授权某个供应商后，它的模型即可使用，因此只有当你需要的模型不在列表中时，才需手动添加，例如新发布或微调的模型。

点击供应商卡片上的 **添加模型**，为模型填写名称和凭据。只提供固定一组模型的供应商不会显示此选项。

<Info>
若添加的模型名称和类型与已有模型完全相同，Dify 会将新密钥附加到该模型上，而不会创建重复的模型。
</Info>

## 选择应用默认使用的模型

未指定模型的应用和节点会回退到工作空间默认值。点击右上角的 **默认模型**，为每项任务分别设置：

- **系统推理模型**：常规 LLM 任务的默认模型。
- **Embedding 模型**：用于知识库内容的索引和检索。
- **Rerank 模型**：按相关性对检索结果重新排序。
- **语音转文本模型**：将音频转换为文本。
- **文本转语音模型**：将文本转换为音频。

## 通过负载均衡将请求分散到多个密钥 <Badge color="blue">Professional</Badge> <Badge color="blue">Team</Badge>

当一个密钥同时处理大量请求时，可能触发供应商的速率限制并开始失败。

使用负载均衡，可将同一模型的请求分散到多个密钥，让任何单个密钥都不会成为瓶颈。Dify 会依次轮询这些密钥，并让触发限制的密钥休息一分钟后再重试。

1. 在列表中找到目标模型，点击 **配置**，选择 **负载均衡**。

2. 点击 **添加凭据**，将密钥加入池中。

    <Tip>
    可将配额更高或速度更快的密钥重复添加，让更多流量流向它。
    </Tip>

    <Frame>
    ![为负载均衡添加凭据](/images/use-dify/workspace/add-load-balancing-credential.png)
    </Frame>

3. 启用至少两个密钥，然后点击 **保存**。使用负载均衡的模型会在列表中显示标识。

<Info>
你可随时切换回单个密钥，负载均衡的配置会保留以备后用。
</Info>
