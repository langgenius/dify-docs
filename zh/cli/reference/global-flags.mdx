---
title: 全局标志
description: 在 difyctl 命令中使用这些标志，控制输出格式、详细程度和重试行为
---

以下标志适用于各个 `difyctl` 命令。`-v` 和 `-h` 通用，`-o` 和 `--http-retry` 则大多数命令支持，但并非全部。某个命令接受哪些标志，参见该命令的 [参考文档](/zh/cli/reference/command-index)。

| 标志 | 作用 |
|:---|:---|
| [`-o, --output <format>`](#输出格式) | 选择输出格式：`json`、`yaml`、`name`、`wide` 或 `text`。 |
| [`-v, --verbose`](#详细模式) | 在 stderr 上开启详细诊断信息。 |
| [`--http-retry <n>`](#http-重试) | 设置幂等请求的重试预算。 |
| [`-h, --help`](/zh/cli/reference/help) | 显示命令帮助，而非执行命令。 |

<Note>
标志始终跟在命令之后：`difyctl get app -o json` 有效；`difyctl -o json get app` 会被当作未知命令。
</Note>

## 输出格式

某个命令支持五种格式中的哪几种，查看它的 `--help` 或参考页上的标志表。重复使用该标志时，以最后一个值为准。

`-o` 还会改变失败信息的呈现方式：在 `-o json` 下，错误会以机器可读的 JSON 对象形式输出到 stderr。

各格式的结构、默认值和通道规则，详见 [输出格式与退出码](/zh/cli/reference/output-formats-and-exit-codes)。

<Note>
在 [`export studio-app`](/zh/cli/reference/apps#导出应用) 上，`-o` 指的是输出文件路径，而非格式选择器。
</Note>

## 详细模式

`-v` 会在 stderr 上增加详细的 HTTP 日志，并在错误输出中包含 `raw_response` 字段（即原始服务端响应，其中的 bearer token 已脱敏）。

当请求失败、而常规错误信息无法说明原因时使用它。

## HTTP 重试

`difyctl` 会在发生瞬时故障时重试幂等请求（GET、PUT、DELETE）：包括网络错误，以及 HTTP 408、413、500、502、503 和 504 响应。POST 请求永不重试。429（触发限流）单独处理，会以退出码 `7` 退出，不计入此预算的重试范围。

默认预算为 3 次尝试，采用指数退避（300 ms 起，每次翻倍，上限 30 s）。使用 `--http-retry 0` 可禁用重试。

[`DIFYCTL_HTTP_RETRY`](/zh/cli/reference/environment-variables) 环境变量设置的是同一预算；两者同时设置时，以标志为准。

## `--workspace` 不是全局标志

`--workspace <id>` 是按命令生效的：它出现在针对工作空间数据的命令上，且仅作用于该次调用。

关于该标志在优先级链（标志、环境变量、存储的默认值）中的位置，参见 [difyctl 如何选择工作空间](/zh/cli/reference/workspaces#difyctl-如何选择工作空间)。
